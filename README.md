# Prompt Vault

Chrome extension for storing and managing AI prompts, synced across devices via Firebase.

Save prompts once. Open them in ChatGPT or Claude with one click. Access them from any Chrome browser signed into your Firebase account.

## What it does

- Store prompts with titles, categories, and tags in Firebase Firestore
- Search across everything — title, text, categories, tags
- One-click copy or direct injection into ChatGPT/Claude
- Quick-save: highlight text on any page, press `Cmd+Shift+S`, it pre-fills the save form
- Star favorites for quick access
- All data stays in your own Firebase project — nothing goes to third parties

## Setup

You need a Firebase project. This takes about 10 minutes.

### 1. Create a Firebase project

1. Go to [console.firebase.google.com](https://console.firebase.google.com/) and create a project
2. Enable **Authentication** (Project settings > Authentication > Sign-in method > Google > Enable)
3. Enable **Firestore** (start in production mode)

### 2. Set Firestore security rules

In Firestore > Rules, replace the default with:

```js
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /prompts/{promptId} {
      allow read, write: if request.auth != null && request.auth.uid == resource.data.userId;
      allow create: if request.auth != null && request.auth.uid == request.resource.data.userId;
    }
  }
}
```

### 3. Get your Firebase config

Project Settings (gear icon) > General > Your apps > web icon (`</>`) > Register app > copy the `firebaseConfig` object.

Create `firebase-config.json` in the extension folder (copy from the example):

```bash
cp firebase-config.example.json firebase-config.json
```

Fill in your values:

```json
{
  "apiKey": "AIza...",
  "authDomain": "your-project.firebaseapp.com",
  "projectId": "your-project",
  "storageBucket": "your-project.appspot.com",
  "messagingSenderId": "123456789",
  "appId": "1:123456789:web:abcdef"
}
```

### 4. Load the extension in Chrome

1. Open `chrome://extensions/`
2. Enable Developer mode (top-right toggle)
3. Click "Load unpacked" and select this folder
4. Pin the extension to your toolbar

Sign in with Google on first use.

## Usage

| Action | How |
|--------|-----|
| Save a prompt | Click the icon, "+ New Prompt" |
| Quick-save selected text | Highlight text, `Cmd+Shift+S` (Mac) / `Ctrl+Shift+S` (Windows) |
| Open in ChatGPT/Claude | Click the ChatGPT or Claude button on any prompt card |
| Copy to clipboard | Click Copy on any card |
| Filter to favorites | Click the star filter at the top |

### Keyboard shortcut

Change it at `chrome://extensions/shortcuts` > find Prompt Vault > edit.

## Customization

Colors are CSS variables in `sidebar.css`:

```css
:root {
  --primary: #2d5d4f;
  --accent: #d97757;
  --bg: #faf8f5;
}
```

Replace `icon16.png`, `icon48.png`, `icon128.png` to change the toolbar icon.

## File structure

```
prompt-vault/
├── manifest.json              # Extension config (Manifest V3)
├── firebase-config.json       # Your Firebase credentials (gitignored)
├── firebase-config.example.json
├── sidebar.html               # Extension UI
├── sidebar.css                # Styles
├── sidebar.js                 # Source (edit this)
├── sidebar-bundle.js          # Compiled output (gitignored, generated by build)
├── background.js              # Service worker
├── content.js                 # Page injection for ChatGPT/Claude
├── build.js                   # esbuild config
└── icon*.png                  # Extension icons
```

## Modifying the code

The extension uses esbuild to bundle the Firebase SDK (can't load remote scripts in Manifest V3).

```bash
npm install
# edit sidebar.js or other source files
npm run build
# reload the extension in chrome://extensions/
```

Changes to `firebase-config.json` don't need a rebuild — it's loaded at runtime.
